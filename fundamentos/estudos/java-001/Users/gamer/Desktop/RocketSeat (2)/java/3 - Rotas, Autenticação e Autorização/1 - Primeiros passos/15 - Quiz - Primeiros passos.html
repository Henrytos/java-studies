Sobre os getters e setters, é correto afirmar:

  • Getters e setters são desnecessários em programação orientada a objetos.
  • Getters e setters são utilizados para acessar e modificar atributos privados de uma classe.
  • Getters e setters violam o princípio de encapsulamento.
  • Getters são exclusivamente usados para obter valores, enquanto setters são usados apenas para definir valores.
  • Getters e setters são conceitos aplicáveis apenas em linguagens de programação específicas.

Sobre o Lombok, é correto afirmar:

  • O Lombok é uma linguagem de programação.
  • O Lombok é exclusivamente utilizado para criar interfaces gráficas.
  • O Lombok ajuda a reduzir a verbosidade do código Java, automatizando a geração de métodos padrão, como getters e setters.
  • O Lombok é uma ferramenta de análise estática de código.
  • O Lombok é incompatível com o paradigma de programação orientada a objetos.

O que é um banco de dados?

  • Uma coleção de dados organizada de forma a facilitar o armazenamento, a recuperação e a análise.
  • Um conjunto de arquivos relacionados entre si, podendo conter registros sobre pessoas, lugares ou informações em geral.
  • Um software que gerencia o armazenamento e o acesso aos dados.
  • Um serviço de armazenamento de dados na nuvem.
  • Um tipo de arquivo que armazena dados de forma estruturada.

No contexto da biblioteca Jakarta Validation em Java, qual é a finalidade principal da anotação @NotNull?

  • @NotNull é utilizada para especificar a presença de uma expressão regular que um campo de um objeto deve atender.
  • A anotação @NotNull é usada para validar se um campo de um objeto está vazio ou não.
  • É uma anotação usada para marcar métodos que não podem ser nulos durante a execução de um programa Java.
  • A anotação @NotNull é usada para garantir que um campo de um objeto não seja nulo.
  • A anotação @NotNull é usada para indicar que um campo de um objeto pode ter valores nulos.

No contexto do Spring Boot, qual é o propósito principal de uma classe anotada com @Controller?

  • A anotação @Controller é usada para definir entidades de persistência em um banco de dados.
  • @Controller é aplicada para criar uma classe que representa uma entidade de negócios no Spring Boot.
  • A anotação @Controller é utilizada para criar controladores que manipulam requisições HTTP, fornecendo uma lógica de apresentação em uma aplicação web.
  • É uma anotação usada para declarar uma classe como um serviço RESTful no Spring Boot.
  • @Controller é aplicada para criar classes que lidam com a segurança e autorização em uma aplicação Spring Boot.

No contexto do Spring Framework, qual é o propósito principal da anotação @Bean?

  • Indicar que uma classe é uma entidade persistente em um banco de dados.
  • Especificar que um método de configuração dentro de uma classe é responsável por criar e configurar um objeto gerenciado pelo container Spring.
  • Desabilitar a injeção de dependências em uma aplicação Spring Boot.
  • Ativar a segurança em uma aplicação Spring Boot.
  • Permitir o acesso irrestrito a todos os métodos da classe.
