No contexto do Domain-Driven Design (DDD), qual é o principal objetivo da definição de um modelo de domínio?

  • Representar a estrutura física da aplicação.
  • Mapear diretamente as entidades do banco de dados.
  • Capturar o conhecimento do negócio e traduzi-lo em um modelo compreensível e aplicável.
  • Fornecer uma implementação detalhada das regras de negócio.
  • Definir as interfaces de usuário da aplicação.

Qual é o propósito do conceito de Bounded Context no Domain-Driven Design (DDD)?

  • Criar uma barreira física entre diferentes partes do código-fonte para facilitar a manutenção.
  • Estabelecer limites geográficos para a implementação do projeto.
  • Delimitar ou criar sub-domínios baseados nas intenções de negócio, permitindo que cada contexto tenha sua própria linguagem ubíqua e abordagem.
  • Definir o escopo do banco de dados utilizado no projeto.
  • Determinar a estrutura de autenticação e autorização do sistema.

Qual é o objetivo do Shared Kernel no mapeamento de contexto, conforme explicado na aula?

  • Estabelecer uma relação cooperativa entre equipes em diferentes contextos.
  • Permitir que várias equipes trabalhem de forma independente em um mesmo domínio.
  • Integrar novos recursos a softwares legados existentes.
  • Identificar equipes responsáveis por diferentes contextos delimitados.
  • Facilitar a comunicação entre equipes downstream e upstream.

Qual é a função principal de um Domain Service?

  • Realizar a persistência das entidades no banco de dados.
  • Agregar objetos imutáveis a outras entidades.
  • Construir adequadamente objetos ou entidades.
  • Atender partes do negócio que não se encaixam em entidades específicas, trabalhando com diversas entidades e realizando persistência através de repositórios.
  • Orquestrar ações disparadas pela camada de apresentação e fornecer DTOs para comunicação entre as demais camadas e para o consumo da camada de apresentação.
