Qual é a finalidade principal do JSON Web Token (JWT) em sistemas de autenticação e autorização?

  • Armazenar dados de usuário, como nome, e-mail e senha.
  • Proteger dados de usuário, como senha e cartão de crédito.
  • Transmitir informações de autenticação e autorização entre sistemas.
  • Autenticação de dois fatores.
  • Autorização de recursos.

Quais são as partes que compõem um token JWT (JSON Web Token)?

  • Header, Payload e Signature.
  • Início, Meio e Fim.
  • Assinatura, Conteúdo e Tempo de Expiração.
  • Identificador, Chave Pública e Chave Privada.
  • Autenticação, Autorização e Validação.

No contexto da autenticação web, qual é o propósito principal do Bearer Token?

  • O Bearer Token é utilizado para armazenar informações sensíveis, como senhas e chaves de criptografia, em cookies de sessão.
  • Representa um método de autenticação onde o usuário fornece um código de barras para autenticar-se em um sistema web.
  • Bearer Token é um tipo de criptomoeda usado para realizar transações seguras em ambientes online.
  • Utilizado para autenticação em APIs, o Bearer Token é um tipo de token de acesso que é enviado no cabeçalho de uma requisição HTTP.
  • O Bearer Token é uma técnica de segurança usada para proteger dados confidenciais armazenados em servidores web.

No contexto do Spring Security, qual é o propósito da anotação `@PreAuthorize("hasRole('COLABORADORES')")`?

  • A anotação @PreAuthorize é utilizada para marcar métodos que só podem ser acessados por colaboradores no sistema.
  • Indica que o método está disponível apenas para usuários autenticados, independentemente da função.
  • Especifica que o método só pode ser acessado por usuários com a função "COLABORADORES".
  • O método com esta anotação será acessível apenas para usuários que possuem um token Bearer.
  • A anotação @PreAuthorize é usada para indicar que um método está disponível para qualquer usuário, independentemente de suas permissões.
