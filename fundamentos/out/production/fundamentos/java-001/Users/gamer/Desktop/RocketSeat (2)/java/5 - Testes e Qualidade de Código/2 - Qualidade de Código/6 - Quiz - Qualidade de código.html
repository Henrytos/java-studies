A refatoração de código é um processo que visa melhorar sua legibilidade e eficiência, tornando-o mais fácil de entender e manter ao longo do tempo.

No contexto do SonarQube, considere as seguintes afirmações sobre as métricas e funcionalidades oferecidas por essa ferramenta. Identifique a opção correta.

  • O SonarQube fornece análises de código estático que ajudam a identificar vulnerabilidades de estilo e layout na aplicação.
  • O SonarQube não suporta a integração com sistemas de controle de versão, tornando difícil rastrear alterações no código ao longo do tempo.
  • O SonarQube não fornece suporte para várias linguagens de programação, limitando sua utilidade a projetos específicos.
  • O SonarQube não oferece relatórios detalhados sobre a cobertura de testes, sendo focado exclusivamente em questões de qualidade de código.
  • O SonarQube permite a definição de perfis de qualidade personalizados, nos quais é possível configurar regras específicas para atender aos padrões de codificação da equipe.

Comentar extensivamente cada linha de código é uma prática recomendada para garantir a qualidade do código, independentemente da situação.

O QG (Quality Gate) no SonarQube é uma medida que avalia se um projeto atende aos critérios de qualidade definidos, e se todos os indicadores de qualidade estão dentro dos limites estabelecidos antes de ser considerado apto para ser implantado em produção.

Qual dos seguintes aspectos é avaliado pelo coverage?

  • Tempo de execução do código.
  • Complexidade do código-fonte.
  • Quantidade de comentários no código.
  • Porcentagem de instruções ou linhas de código executadas durante os testes.
  • Número total de funções no código.
